# 要件定義書 — みせ勤

## 1. プロダクト概要

**アプリ名**: みせ勤 (Miseki)
**対象**: 複数店舗を運営する小規模事業者・飲食店・コンカフェ・バーなど
**目的**: 深夜営業・日またぎ勤務・複数店舗所属に対応した継続運用可能な勤怠管理ツール

## 2. ステークホルダー

| 役割 | 説明 |
|------|------|
| 組織オーナー | 組織を作成した最初のユーザー。最高権限を持つ |
| 管理者 | オーナーから招待された管理者。店舗スコープを制限可能 |
| スタッフ | 勤怠記録の当事者。打刻と自分の勤怠確認が主な操作 |
| 外部システム | APIキーを使った外部連携システム |

## 3. 機能要件

### 3.1 認証・アカウント管理
- メールアドレス + パスワードによる登録・ログイン
- メールアドレス確認（メール送信）
- パスワード再設定（メール送信）
- セッション管理（httpOnly Cookie）
- 将来的にOAuth（Google等）追加可能な構造
- ログイン試行回数制限

### 3.2 組織管理
- ユーザーが最初に「組織」を作成する（UIは「お店・グループ」として表示可）
- 組織オーナーは1人以上必須（0人にならない制約）
- オーナーは自分が唯一のオーナーであれば権限を下げられない
- 複数組織への所属（組織切替機能）

### 3.3 店舗管理
- 組織内に複数店舗を作成
- 店舗ごとのタイムゾーン・営業日切替時刻を設定
- 店舗は物理削除せず無効化（過去勤怠との関連を保持）
- 店舗専用打刻URLの発行・再発行・無効化・有効期限設定
- QRコード表示

### 3.4 スタッフ管理
- スタッフはメール招待（招待→在籍の状態遷移）
- 1スタッフが複数店舗に所属可能
- 在籍状態：招待中 / 在籍 / 休職 / 退職 / 利用停止
- 物理削除不可
- 時給は適用期間付き履歴として管理（上書き不可）
- 交通費は店舗所属ごとに設定
- 打刻用PIN（ハッシュ保存）

### 3.5 打刻機能
- 店舗専用URLから管理者ログイン不要で打刻
- スタッフ選択 → PIN入力 → 打刻操作
- 状態遷移：未出勤 → 勤務中 → 休憩中 → 勤務中 → 退勤済み
- 不正な状態遷移は防止（UI + サーバー両方で検証）
- 休憩中退勤は確認ダイアログを表示し退勤時刻と同時刻で休憩終了を自動記録
- PIN連続失敗にレート制限
- 打刻後の完了画面（数秒後にスタッフ選択へ戻る）
- 共用端末考慮（他人の情報を画面に残さない）

### 3.6 勤怠管理
- 日またぎ勤務：勤務日は営業日切替時刻で決定（午前0時分割不可）
- 実打刻時刻の保全（上書き禁止）
- 異常検出：退勤漏れ、休憩終了漏れ、逆転時刻、長時間勤務、重複打刻、複数店舗同時勤務
- 管理者による勤怠修正（修正前値を保全、理由必須、監査ログ記録）
- スタッフによる修正申請（申請中 / 承認済 / 却下 / 取消）
- 締め処理・勤怠ロック

### 3.7 監査ログ
- 全重要操作を記録（編集・削除不可）
- 記録項目：操作者、日時、種別、対象、変更前後、理由、IP、UA

### 3.8 CSVエクスポート
- UTF-8 BOM付き、Excel文字化け対策
- 大量データでもストリーミング出力
- 出力時に監査ログ記録

### 3.9 外部REST API
- APIキー認証（発行時1回のみ表示、DBはハッシュ保存）
- 組織・店舗スコープ
- レート制限
- OpenAPI仕様書 + Swagger UI
- バージョニング（/api/v1/）

### 3.10 通知
- Web通知（アプリ内）
- メール通知
- 通知種別：招待、修正申請作成・承認・却下、退勤漏れ、長時間勤務、APIキー発行・無効化

## 4. 非機能要件

### 4.1 セキュリティ
- OWASP Top 10 対策
- 全DBクエリで組織スコープを明示（IDOR防止）
- フロントエンドの表示制御だけでなくサーバー側で認可検証
- セキュアCookie（httpOnly, SameSite）
- 秘密情報は環境変数管理

### 4.2 パフォーマンス
- 勤怠一覧のページネーション
- 大量CSVのストリーミング
- Server Componentsを適切に使用

### 4.3 アクセシビリティ
- WCAG 2.2 AA相当
- セマンティックHTML
- キーボード操作
- aria属性
- コントラスト比
- reduced-motion対応

### 4.4 レスポンシブ
- PC（1280px+）：管理画面
- モバイル（390px〜）：打刻画面・スタッフ画面優先

## 5. データ保全原則
- 物理削除原則禁止
- 実打刻イベント上書き禁止
- 修正データと元データの分離
- DB制約とトランザクションの活用
- 二重送信防止（打刻の冪等性）

## 6. MVP範囲外（次フェーズ）
- シフト管理
- 有給休暇管理
- 給与明細発行
- 深夜・残業自動計算（データ構造は準備済み）
- 高度なダッシュボード・グラフ
- CSVインポート
- Webhook
- 位置情報打刻制限
- 外部サービス連携（LINE、Slack、マネーフォワード等）
